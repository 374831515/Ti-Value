Build console project

Problem: Unresolved external symbol _WinMain @ 16, referenced in function ___tmainCRTStartup
The real reason for this problem is that c language can not find the appropriate program entry function in running. Under normal circumstances, if it is a windows program, then WinMain is an entry function; if it is a dos console program, then main is the entry function. If the entrance function is not properly specified, it is clear that c language can not find a matching function in runing, so it will report an error.

Solution: Linker -> All Options -> Subsystem   Select Console

问题：无法解析的外部符号 _WinMain@16，该符号在函数 ___tmainCRTStartup 中被引用
产生这个问题的真正原因是c语言运行时找不到适当的程序入口函数，一般情况下，如果是windows程序，那么WinMain是入口函数，如果是dos控制台程序，那么main是入口函数，而如果入口函数指定不当，很显然c语言运行时找不到配合函数，它就会报告错误。

解决： 链接器-〉所有选项-〉子系统  选择控制台
